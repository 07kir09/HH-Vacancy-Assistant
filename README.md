# HH Vacancy Assistant

Локальный ассистент для поиска вакансий на hh.ru через официальный API.

Проект умеет:

- искать вакансии по ключевым словам и фильтрам;
- оценивать релевантность вакансии по score;
- генерировать сопроводительное письмо под конкретную вакансию;
- сохранять черновики откликов в локальную SQLite-базу;
- запускать несколько независимых стратегий поиска за один скан;
- объяснять score вакансии понятными причинами;
- разделять вакансии на `Рекомендовано` и `На проверке`;
- редактировать и сохранять сопроводительные письма перед откликом;
- проверять качество письма: персонализацию, совпадения с требованиями,
  подтверждение фактами из профиля, длину и неподтвержденные навыки;
- вести локальную статистику и обратную связь по вакансиям;
- по желанию присылать Telegram-дайджест новых рекомендованных вакансий;
- открывать официальную страницу отклика на hh.ru;
- работать с несколькими локальными пользователями;
- парсить PDF/DOCX/TXT/MD резюме и заполнять профиль кандидата;
- показывать качество распознавания резюме и сырой извлеченный текст.

Важно: приложение не отправляет отклики через API. Поддержка API для соискателей прекращена, поэтому отклик отправляется вручную на сайте hh.ru. Приложение готовит вакансию, письмо и ссылку.

## Как работает процесс

1. Ты создаешь приложение на dev.hh.ru и получаешь `HH_CLIENT_ID` и `HH_CLIENT_SECRET`.
2. Запускаешь этот проект локально.
3. Открываешь веб-интерфейс.
4. Создаешь пользователя.
5. Загружаешь резюме.
6. Проверяешь качество распознавания и сырой текст.
7. Открываешь профиль, правишь ошибки и подтверждаешь профиль.
8. Вставляешь `HH_CLIENT_ID` и `HH_CLIENT_SECRET`.
9. Вводишь контактный email для HH User-Agent.
10. Нажимаешь `Проверить доступ`.
11. Нажимаешь `Найти вакансии`.
12. Открываешь вкладку `Черновики`.
13. Выбираешь вакансию, копируешь письмо и открываешь отклик на hh.ru.

## Требования

- Python 3.11 или новее
- Аккаунт на hh.ru
- Зарегистрированное приложение на dev.hh.ru
- macOS/Linux/Windows с терминалом

Проверить Python:

```bash
python3 --version
```

## Установка

Склонируй репозиторий:

```bash
git clone https://github.com/07kir09/HH-Vacancy-Assistant.git
cd HH-Vacancy-Assistant
```

Создай виртуальное окружение:

```bash
python3 -m venv .venv
```

Активируй его:

```bash
source .venv/bin/activate
```

На Windows:

```bash
.venv\Scripts\activate
```

Установи зависимости:

```bash
pip install -r requirements.txt
```

## Как получить HH_CLIENT_ID и HH_CLIENT_SECRET

1. Открой `https://dev.hh.ru`.
2. Войди в аккаунт hh.ru.
3. Перейди в регистрацию нового приложения.
4. Заполни заявку.

Пример названия приложения:

```text
HH Vacancy Assistant
```

Пример Redirect URI:

```text
http://localhost:8787/callback
```

Если форма просит описание приложения, можно написать:

```text
Приложение используется для поиска и анализа вакансий на hh.ru через официальный API.
Оно получает application token, выполняет поиск вакансий, получает подробную информацию
о вакансиях, оценивает релевантность найденных вакансий и помогает пользователю
подготовить черновик сопроводительного письма.

Приложение не парсит сайт hh.ru, не хранит логин и пароль пользователя, не отправляет
отклики автоматически и не использует методы API для соискателей.
```

Используемые методы API:

```text
POST /token
GET /vacancies
GET /vacancies/{vacancy_id}
```

После одобрения заявки hh.ru выдаст:

```text
HH_CLIENT_ID
HH_CLIENT_SECRET
```

Их нужно будет вставить в веб-интерфейс приложения.

## Куда нажимать в интерфейсе

### 1. Создать пользователя

Слева в поле `Новый пользователь` введи короткое имя:

```text
kirill
```

или:

```text
friend
```

Нажми `Создать`.

Для пользователя будет создана локальная папка:

```text
data/users/<user>/
```

Там будут храниться его профиль, настройки поиска, локальная база и HH-ключи.

Чтобы убрать ненужный профиль, нажми `Удалить` справа от его имени в левом списке
и подтверди действие. Удаляются только локальные данные этого пользователя:
резюме, ключи, настройки и история откликов. Восстановить их после удаления нельзя.

### 2. Загрузить резюме

Во вкладке `1. Настройка` найди блок `Резюме кандидата`.

Нажми `Выберите файл` и выбери резюме:

- PDF
- DOCX
- TXT
- MD

Потом нажми:

```text
Загрузить резюме
```

После этого приложение:

- извлечет текст из резюме;
- найдет имя, контакты, навыки, роли, проекты;
- заполнит профиль;
- обновит ключевые слова поиска;
- покажет отчет качества распознавания;
- покажет сырой текст, который удалось извлечь из файла.

Если score распознавания низкий, проверь сырой текст. Если там мало текста или текст идет в странном порядке, лучше загрузить DOCX/TXT или другой PDF с текстовым слоем. OCR для сканов пока не реализован.

### 3. Вставить HH Client ID и Client Secret

Во вкладке `1. Настройка` найди блок `Доступ к HH API`.

Вставь:

```text
HH_CLIENT_ID
HH_CLIENT_SECRET
```

В поле `Контактный email для HH User-Agent` укажи реальный email, по которому с тобой можно связаться. Не используй `example.com`, тестовые адреса или пустое значение. HH может отклонять запросы с плохим User-Agent.

Нажми:

```text
Сохранить ключи
```

Потом нажми:

```text
Проверить доступ
```

Если все хорошо, появится сообщение, что application token сохранен.

### 4. Проверить профиль

Открой вкладку:

```text
2. Профиль
```

Там будет JSON-профиль кандидата. Можно поправить:

- `name`
- `profile_reviewed`
- `target_roles`
- `desired_salary`
- `skills`
- `preferred_keywords`
- `cover_letter_facts`

После правок нажми:

```text
Сохранить и подтвердить профиль
```

Без этого шага поиск из веб-интерфейса не запустится. Это сделано специально, чтобы не искать вакансии по неправильно распознанному резюме.

### 5. Настроить поиск

Открой вкладку:

```text
3. Поиск
```

Заполни обычную анкету и нажми:

```text
Сохранить настройки
```

В анкете можно задать:

- роли и ключевые слова;
- города;
- желаемую зарплату и необходимость указанной вилки;
- формат работы: любой, удаленка/гибрид или только офис;
- готовность к релокации;
- свой уровень английского;
- требуемый опыт и тип занятости;
- стоп-компании и стоп-слова;
- исключение вакансий кадровых агентств.

Стоп-компания сравнивается с названием работодателя. Агентства определяются по
явным словам в названии или описании, например `кадровое агентство`,
`recruitment agency` или `подбор персонала`. Это эвристика: если конкретная
компания нежелательна, добавь ее в стоп-лист вручную.

Блок `Расширенные настройки YAML` оставлен для редких случаев: нескольких
стратегий, лимитов, Telegram и ручной настройки score. После изменения YAML
нажми `Сохранить YAML`, чтобы анкета снова показала актуальные значения.

### Несколько стратегий поиска

Одна стратегия полезна для одной группы ролей. Например, продуктовую аналитику
можно оценивать строже, чем общий поиск по Data Analyst. Добавь в блок `search`:

```yaml
strategies:
  - name: "Продуктовая аналитика"
    keywords:
      - "Product Analyst"
      - "Product Data Analyst"
    filters:
      min_score: 75

  - name: "Аналитика данных и BI"
    keywords:
      - "Data Analyst"
      - "BI Analyst"
    enabled: true
```

Поля внутри стратегии переопределяют общие параметры поиска только для этой
стратегии. Обычные `search.keywords` продолжают работать, если стратегии не нужны.

Уровень автоматической рекомендации задается отдельно:

```yaml
filters:
  min_score: 65
  recommended_score: 80
```

Вакансии с score от `recommended_score` попадут в `Рекомендовано`; остальные,
которые прошли `min_score`, будут в `На проверке`.

### 6. Найти вакансии

Во вкладке `3. Поиск` нажми:

```text
Найти вакансии
```

Приложение:

- получит вакансии из HH API;
- скачает подробное описание каждой вакансии;
- посчитает score;
- отфильтрует нерелевантные вакансии;
- сгенерирует сопроводительное письмо;
- проверит качество письма;
- сохранит черновики.

### 7. Открыть черновики

Открой вкладку:

```text
4. Черновики
```

Нажми `Выбрать` рядом с вакансией.

Справа появится сопроводительное письмо и блок проверки качества.

Дальше:

1. Проверь блок качества письма.
2. При необходимости отредактируй текст и нажми `Проверить письмо`.
3. Нажми `Копировать`.
4. Нажми `Открыть на HH`.
5. На hh.ru вставь письмо в форму.
6. Отправь отклик вручную.
7. Вернись в приложение.
8. Нажми `Отправил`.

Проверка письма показывает один из статусов:

- `Готово к отклику` — письмо достаточно персонализировано и не содержит
  критичных предупреждений.
- `Нужна быстрая проверка` — письмо можно использовать, но лучше глазами
  проверить предупреждения.
- `Лучше доработать` — письмо слабое или слишком шаблонное. При нажатии
  `Открыть на HH` приложение попросит дополнительное подтверждение.

Кнопка `Открыть на HH` сначала сохраняет текущий текст письма, пересчитывает
качество, копирует письмо в буфер обмена и открывает страницу отклика.

Если вакансия не подходит, нажми:

```text
Не подходит
```

Письмо можно отредактировать прямо в правой части экрана и нажать `Сохранить`.
Кнопки `Подходит` и `Не подходит` сохраняют твою оценку и заметку в локальной
истории; `Не подходит` также переносит вакансию в пропущенные. Во вкладке `Статистика` видно количество рекомендаций, черновиков,
отправленных откликов и таких оценок.

## Автоматический поиск по расписанию

Для запуска поиска каждые 12 часов открой отдельный терминал и выполни:

```bash
python main.py --user friend schedule --interval-minutes 720
```

Команда работает, пока терминал открыт. Для одной проверки без ожидания:

```bash
python main.py --user friend schedule --once
```

Минимальный интервал — 5 минут. Не ставь слишком частый запуск: он не улучшает
качество поиска и создает лишнюю нагрузку на HH API.

## Telegram-уведомления (необязательно)

Создай бота через `@BotFather`, напиши ему любое сообщение и получи свой `chat_id`.
В локальном `data/users/<user>/config.yaml` добавь:

```yaml
notifications:
  telegram:
    enabled: true
    bot_token: "123456:replace-with-real-token"
    chat_id: "123456789"
```

После каждого сканирования бот пришлет только новые вакансии из категории
`Рекомендовано`. Токен хранится в локальной пользовательской папке, которая
исключена из Git. Не добавляй его в общий `config.yaml` репозитория.

## CLI-команды

Веб-интерфейс удобнее, но можно пользоваться и через терминал.

Импорт резюме:

```bash
python main.py import-resume "/path/to/resume.pdf" --user friend
```

Сохранить HH credentials:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..."
```

Лучше сразу указать контактный email для HH User-Agent:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..." --contact-email "name@your-domain.ru"
```

Получить application token:

```bash
python main.py --user friend app-token
```

Найти вакансии:

```bash
python main.py --user friend scan
```

Запустить полуавтоматический агент:

```bash
python main.py --user friend agent
```

Важно: для команд `app-token`, `scan`, `review`, `agent` параметр `--user friend` ставится до названия команды.

Для `schedule` правило такое же:

```bash
python main.py --user friend schedule --interval-minutes 720
```

## Где хранятся данные

Локальные данные пользователя:

```text
data/users/<user>/
```

Внутри:

```text
config.yaml
resume_profile.json
credentials.json
job_apply_bot.db
uploads/
```

Что это значит:

- `config.yaml` — настройки поиска;
- `resume_profile.json` — профиль из резюме;
- `credentials.json` — HH Client ID и Client Secret;
- `job_apply_bot.db` — SQLite-база с токенами и черновиками;
- `uploads/` — загруженные резюме.

В базе дополнительно хранятся стратегия поиска, причины score, статус вакансии,
отредактированное письмо, отметка открытия страницы и твоя обратная связь.

Папка `data/users/` добавлена в `.gitignore`, чтобы личные данные, резюме, токены и базы не попали в GitHub.
Корневой `resume_profile.json` также исключен из Git. В репозитории остается только
безопасный [шаблон](resume_profile.example.json); обычная работа проекта должна идти
через отдельного пользователя в `data/users/<user>/`.

## Как другу установить или обновить проект

Лучший вариант - отдельный клон репозитория, а не копирование всей твоей папки:

```bash
git clone https://github.com/07kir09/HH-Vacancy-Assistant.git
cd HH-Vacancy-Assistant
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py web
```

В веб-интерфейсе друг создает свой ID пользователя, например `ivan`, загружает
свое резюме и вставляет только свои HH Client ID и Client Secret. Папку
`data/users/` между компьютерами передавать не нужно: в ней находятся личные
резюме, токены, история поиска и черновики.

Чтобы обновить уже установленный клон:

```bash
git pull
source .venv/bin/activate
pip install -r requirements.txt
python main.py web
```

Если раньше проект был передан архивом или копированием папки, проще сделать
свежий клон по инструкции выше. Если в левой панели остались чужие пользователи,
удали их кнопкой `Удалить` на компьютере друга и создай новый профиль. Для CLI
каждая команда теперь требует `--user`, например:

```bash
python main.py --user ivan scan
```

После обновления до новой версии каждому пользователю нужно один раз снова
загрузить свое резюме во вкладке «Настройка». Это очистит старые черновики этого
пользователя и заново построит поиск только по его ролям и навыкам. HH-ключи при
этом сохраняются.

## Безопасность

Нельзя публиковать:

- `HH_CLIENT_SECRET`;
- `credentials.json`;
- `job_apply_bot.db`;
- загруженные резюме;
- корневой `resume_profile.json`.
- личные контакты кандидата без согласия.

Если секрет случайно попал в публичный репозиторий, его нужно перевыпустить в кабинете dev.hh.ru.

## Что делать, если ошибка

### `HH_CLIENT_ID and HH_CLIENT_SECRET are required`

Не сохранены ключи. В веб-интерфейсе открой `1. Настройка`, вставь оба значения и нажми `Сохранить ключи`.

### `OAuth token request failed`

Возможные причины:

- заявка приложения еще не одобрена;
- неверный `HH_CLIENT_ID`;
- неверный `HH_CLIENT_SECRET`;
- в dev.hh.ru не настроено приложение.

### `Bad User-Agent header`

HH отклонил запрос из-за плохого `User-Agent`. Обычно это происходит, если в конфиге остался шаблонный email, `example.com` или старое значение `job_apply_bot`.

Что сделать:

1. Открой веб-интерфейс.
2. Перейди во вкладку `1. Настройка`.
3. В блоке `Доступ к HH API` заполни `Контактный email для HH User-Agent`.
4. Нажми `Сохранить ключи`.
5. Снова нажми `Проверить доступ` или `Найти вакансии`.

Через CLI:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..." --contact-email "name@your-domain.ru"
```

### `No draft applications found`

Сначала запусти поиск:

```bash
python main.py --user friend scan
```

или нажми `Найти вакансии` в веб-интерфейсе.

### PDF загрузился, но профиль пустой

В PDF может не быть текстового слоя. Попробуй:

- DOCX;
- TXT;
- другой PDF;
- скопировать текст резюме в `.txt` и загрузить его.

### Поиск просит подтвердить профиль

Открой вкладку `2. Профиль`, проверь распознанные данные и нажми:

```text
Сохранить и подтвердить профиль
```

После этого вернись во вкладку `3. Поиск` и снова нажми `Найти вакансии`.

## Ограничения

- Приложение не отправляет отклики автоматически через API.
- Приложение не работает с резюме соискателя через HH API.
- Приложение не работает с переговорами/откликами соискателя через HH API.
- Для поиска и просмотра вакансий используется application token.
- PDF-резюме корректно парсится только если в нем есть текстовый слой.
- Сканы и картинки без OCR не распознаются.

## Быстрый старт

```bash
git clone https://github.com/07kir09/HH-Vacancy-Assistant.git
cd HH-Vacancy-Assistant
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py web
```

Потом открыть:

```text
http://127.0.0.1:8787
```
