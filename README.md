# HH Vacancy Assistant

Локальный ассистент для поиска вакансий на hh.ru через официальный API.

Проект умеет:

- искать вакансии по ключевым словам и фильтрам;
- оценивать релевантность вакансии по score;
- генерировать сопроводительное письмо под конкретную вакансию;
- сохранять черновики откликов в локальную SQLite-базу;
- открывать официальную страницу отклика на hh.ru;
- работать с несколькими локальными пользователями;
- парсить PDF/DOCX/TXT/MD резюме и заполнять профиль кандидата;
- показывать качество распознавания резюме и сырой извлеченный текст.

Важно: приложение не отправляет отклики через API. Поддержка API для соискателей прекращена, поэтому отклик отправляется вручную на сайте hh.ru. Приложение готовит вакансию, письмо и ссылку.

## Как работает процесс

1. Ты создаешь приложение на dev.hh.ru и получаешь `HH_CLIENT_ID` и `HH_CLIENT_SECRET`.
2. Запускаешь этот проект локально.
3. Открываешь веб-интерфейс.
4. Создаешь пользователя.
5. Загружаешь резюме.
6. Проверяешь качество распознавания и сырой текст.
7. Открываешь профиль, правишь ошибки и подтверждаешь профиль.
8. Вставляешь `HH_CLIENT_ID` и `HH_CLIENT_SECRET`.
9. Вводишь контактный email для HH User-Agent.
10. Нажимаешь `Проверить доступ`.
11. Нажимаешь `Найти вакансии`.
12. Открываешь вкладку `Черновики`.
13. Выбираешь вакансию, копируешь письмо и открываешь отклик на hh.ru.

## Требования

- Python 3.11 или новее
- Аккаунт на hh.ru
- Зарегистрированное приложение на dev.hh.ru
- macOS/Linux/Windows с терминалом

Проверить Python:

```bash
python3 --version
```

## Установка

Склонируй репозиторий:

```bash
git clone https://github.com/07kir09/HH-Vacancy-Assistant.git
cd HH-Vacancy-Assistant
```

Создай виртуальное окружение:

```bash
python3 -m venv .venv
```

Активируй его:

```bash
source .venv/bin/activate
```

На Windows:

```bash
.venv\Scripts\activate
```

Установи зависимости:

```bash
pip install -r requirements.txt
```

## Как получить HH_CLIENT_ID и HH_CLIENT_SECRET

1. Открой `https://dev.hh.ru`.
2. Войди в аккаунт hh.ru.
3. Перейди в регистрацию нового приложения.
4. Заполни заявку.

Пример названия приложения:

```text
HH Vacancy Assistant
```

Пример Redirect URI:

```text
http://localhost:8787/callback
```

Если форма просит описание приложения, можно написать:

```text
Приложение используется для поиска и анализа вакансий на hh.ru через официальный API.
Оно получает application token, выполняет поиск вакансий, получает подробную информацию
о вакансиях, оценивает релевантность найденных вакансий и помогает пользователю
подготовить черновик сопроводительного письма.

Приложение не парсит сайт hh.ru, не хранит логин и пароль пользователя, не отправляет
отклики автоматически и не использует методы API для соискателей.
```

Используемые методы API:

```text
POST /token
GET /vacancies
GET /vacancies/{vacancy_id}
```

После одобрения заявки hh.ru выдаст:

```text
HH_CLIENT_ID
HH_CLIENT_SECRET
```

Их нужно будет вставить в веб-интерфейс приложения.

## Запуск веб-интерфейса

Из папки проекта:

```bash
source .venv/bin/activate
python main.py web
```

Открой в браузере:

```text
http://127.0.0.1:8787
```

Если порт занят, можно запустить на другом:

```bash
python main.py web --port 8790
```

Тогда открыть:

```text
http://127.0.0.1:8790
```

## Куда нажимать в интерфейсе

### 1. Создать пользователя

Слева в поле `Новый пользователь` введи короткое имя:

```text
kirill
```

или:

```text
friend
```

Нажми `Создать`.

Для пользователя будет создана локальная папка:

```text
data/users/<user>/
```

Там будут храниться его профиль, настройки поиска, локальная база и HH-ключи.

### 2. Загрузить резюме

Во вкладке `1. Настройка` найди блок `Резюме кандидата`.

Нажми `Выберите файл` и выбери резюме:

- PDF
- DOCX
- TXT
- MD

Потом нажми:

```text
Загрузить резюме
```

После этого приложение:

- извлечет текст из резюме;
- найдет имя, контакты, навыки, роли, проекты;
- заполнит профиль;
- обновит ключевые слова поиска;
- покажет отчет качества распознавания;
- покажет сырой текст, который удалось извлечь из файла.

Если score распознавания низкий, проверь сырой текст. Если там мало текста или текст идет в странном порядке, лучше загрузить DOCX/TXT или другой PDF с текстовым слоем. OCR для сканов пока не реализован.

### 3. Вставить HH Client ID и Client Secret

Во вкладке `1. Настройка` найди блок `Доступ к HH API`.

Вставь:

```text
HH_CLIENT_ID
HH_CLIENT_SECRET
```

В поле `Контактный email для HH User-Agent` укажи реальный email, по которому с тобой можно связаться. Не используй `example.com`, тестовые адреса или пустое значение. HH может отклонять запросы с плохим User-Agent.

Нажми:

```text
Сохранить ключи
```

Потом нажми:

```text
Проверить доступ
```

Если все хорошо, появится сообщение, что application token сохранен.

### 4. Проверить профиль

Открой вкладку:

```text
2. Профиль
```

Там будет JSON-профиль кандидата. Можно поправить:

- `name`
- `profile_reviewed`
- `target_roles`
- `desired_salary`
- `skills`
- `preferred_keywords`
- `cover_letter_facts`

После правок нажми:

```text
Сохранить и подтвердить профиль
```

Без этого шага поиск из веб-интерфейса не запустится. Это сделано специально, чтобы не искать вакансии по неправильно распознанному резюме.

### 5. Настроить поиск

Открой вкладку:

```text
3. Поиск
```

Главные поля:

```yaml
search:
  keywords:
    - "Data Analyst"
    - "Product Analyst"
    - "SQL"
    - "Python"
```

Зарплата:

```yaml
desired_salary: 150000
```

Минимальный score вакансии:

```yaml
filters:
  min_score: 65
```

Если хочешь больше вакансий, уменьши score:

```yaml
min_score: 55
```

Если хочешь только более точные вакансии, увеличь:

```yaml
min_score: 75
```

После изменений нажми:

```text
Сохранить поиск
```

### 6. Найти вакансии

Во вкладке `3. Поиск` нажми:

```text
Найти вакансии
```

Приложение:

- получит вакансии из HH API;
- скачает подробное описание каждой вакансии;
- посчитает score;
- отфильтрует нерелевантные вакансии;
- сгенерирует сопроводительное письмо;
- сохранит черновики.

### 7. Открыть черновики

Открой вкладку:

```text
4. Черновики
```

Нажми `Выбрать` рядом с вакансией.

Справа появится сопроводительное письмо.

Дальше:

1. Нажми `Копировать письмо`.
2. Нажми `Открыть отклик`.
3. На hh.ru вставь письмо в форму.
4. Отправь отклик вручную.
5. Вернись в приложение.
6. Нажми `Отметить отправленным`.

Если вакансия не подходит, нажми:

```text
Пропустить
```

## CLI-команды

Веб-интерфейс удобнее, но можно пользоваться и через терминал.

Импорт резюме:

```bash
python main.py import-resume "/path/to/resume.pdf" --user friend
```

Сохранить HH credentials:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..."
```

Лучше сразу указать контактный email для HH User-Agent:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..." --contact-email "name@your-domain.ru"
```

Получить application token:

```bash
python main.py --user friend app-token
```

Найти вакансии:

```bash
python main.py --user friend scan
```

Запустить полуавтоматический агент:

```bash
python main.py --user friend agent
```

Важно: для команд `app-token`, `scan`, `review`, `agent` параметр `--user friend` ставится до названия команды.

## Где хранятся данные

Локальные данные пользователя:

```text
data/users/<user>/
```

Внутри:

```text
config.yaml
resume_profile.json
credentials.json
job_apply_bot.db
uploads/
```

Что это значит:

- `config.yaml` — настройки поиска;
- `resume_profile.json` — профиль из резюме;
- `credentials.json` — HH Client ID и Client Secret;
- `job_apply_bot.db` — SQLite-база с токенами и черновиками;
- `uploads/` — загруженные резюме.

Папка `data/users/` добавлена в `.gitignore`, чтобы личные данные, резюме, токены и базы не попали в GitHub.

## Безопасность

Нельзя публиковать:

- `HH_CLIENT_SECRET`;
- `credentials.json`;
- `job_apply_bot.db`;
- загруженные резюме;
- личные контакты кандидата без согласия.

Если секрет случайно попал в публичный репозиторий, его нужно перевыпустить в кабинете dev.hh.ru.

## Что делать, если ошибка

### `HH_CLIENT_ID and HH_CLIENT_SECRET are required`

Не сохранены ключи. В веб-интерфейсе открой `1. Настройка`, вставь оба значения и нажми `Сохранить ключи`.

### `OAuth token request failed`

Возможные причины:

- заявка приложения еще не одобрена;
- неверный `HH_CLIENT_ID`;
- неверный `HH_CLIENT_SECRET`;
- в dev.hh.ru не настроено приложение.

### `Bad User-Agent header`

HH отклонил запрос из-за плохого `User-Agent`. Обычно это происходит, если в конфиге остался шаблонный email, `example.com` или старое значение `job_apply_bot`.

Что сделать:

1. Открой веб-интерфейс.
2. Перейди во вкладку `1. Настройка`.
3. В блоке `Доступ к HH API` заполни `Контактный email для HH User-Agent`.
4. Нажми `Сохранить ключи`.
5. Снова нажми `Проверить доступ` или `Найти вакансии`.

Через CLI:

```bash
python main.py set-credentials --user friend --client-id "..." --client-secret "..." --contact-email "name@your-domain.ru"
```

### `No draft applications found`

Сначала запусти поиск:

```bash
python main.py --user friend scan
```

или нажми `Найти вакансии` в веб-интерфейсе.

### PDF загрузился, но профиль пустой

В PDF может не быть текстового слоя. Попробуй:

- DOCX;
- TXT;
- другой PDF;
- скопировать текст резюме в `.txt` и загрузить его.

### Поиск просит подтвердить профиль

Открой вкладку `2. Профиль`, проверь распознанные данные и нажми:

```text
Сохранить и подтвердить профиль
```

После этого вернись во вкладку `3. Поиск` и снова нажми `Найти вакансии`.

## Ограничения

- Приложение не отправляет отклики автоматически через API.
- Приложение не работает с резюме соискателя через HH API.
- Приложение не работает с переговорами/откликами соискателя через HH API.
- Для поиска и просмотра вакансий используется application token.
- PDF-резюме корректно парсится только если в нем есть текстовый слой.
- Сканы и картинки без OCR не распознаются.
